# Семинары программы "Технологии БАС в образовании"

## Подготовка окружения

1. Создаем среду:

```bash
python3 -m venv seminar_env
```

2. Активируем её:

```bash
source ./seminar_env/bin/activate
```

3. Скачиваем в неё библиотеки:

```bash
pip install -r requirements.txt
```

## Запуск нужной карты в симуляторе

1. Нужно находиться в корневой папке, т.е. в папке `seminarU2035`
2. Из этой папки пишем в терминал:
```bash
./start_sim.sh <номер_темы_семинара>
```

## Форматирование кода

Для удобства в корневой папке репозитория лежат файлы `formatter.sh` (запускает форматирование всех файлов) и `ruff.toml` (настройки форматирования).

Для использования выполните команду (с активированной средой):

```bash
./formatter.sh
```

## Задания к семинарам

### Тема 4.3. "Использование симуляторов в образовательном процессе"

1. Запустить в симуляторе UAVProf с помощью `starter` или `starter2` карту `race_stud_test_settings.json`
2. Запустить QGroundControl и добавить в нем подключение к дрону в симуляторе

**Результат**: Снимок экрана, на котором видны запущенные симулятор и QGC

### Тема 4.4. "Программирование полетного задания БВС"

1. Запустить в симуляторе UAVProf с помощью `starter` или `starter2` карту `formation_stud_test_settings.json`
2. Запустить QGroundControl и добавить в нем подключение к дрону в симуляторе
3. Построить в QGC миссию, в которой дрон взлетает (высоту можете выбрать самостоятельно) и пролетает около каждого из выходов

**Результат**: Снимок экрана, на котором видна составленная миссия

### Тема 5.4. "Работа с GPS и геоданными"

1. Повторить пролет по кубам (аналогично приведенному в блокноте), использовав формулу для эллипсоида
2. Добавить в формулы расчета расстояния изменение высоты (то есть использовать аргументы `alt1` и `alt2`)
3. Рассчитать расстояние между кубами на одной диагонали

**Результат**: Снимок экрана, на котором виден написанный код и рассчитанное расстояние между кубами

### Тема 6.4. "Логирование и анализ полетных данных"

1. В файле `home.py` создать логгер с именем `AnotherLogger`
2. Добавить 3 обработчика:
    1. Выводит в консоль (`sys.stdout`) все сообщения уровня `logging.WARNING` и выше
    2. Выводит в файл `info.log` (оставляя старое содержимое) все сообщения уровня `logging.INFO` и выше
    3. Выводит в файл `debug.log` (затирая старое содержимое) все сообщения уровня `logging.DEBUG` и выше
3. Для первого обработчика добавить форматирование вида `"%(asctime)s - %(levelname)s - %(name)s - %(message)s"`
4. Для третьего обработчика добавить фильтр, который оставляет только сообщения уровня `logging.DEBUG`
5. Запустить скрипт `home.py`

**Результат**: Снимок экрана, на котором видны логи в консоли и каждом из файлов
